每日一题：循环数组拆炸弹，C++取模运算踩坑

2022/9/24 算法 每日一题 力扣 模拟 滑动窗口

碰到的坑

在奇怪的地方浪费了一小时才搞明白问题。

坑一：C++的零向取余

C++的负数取余方式不同于数学上的取余，

例如 i%n，在数学上和PYTHON中：

-5%7 = -5 - （7*-1）= 2 
即7乘数之后小于-5，这是负向取余。

而在C++中，采用的是零向取余，即7乘数之后要大于-5

所以-5%7= -5 -(7* 0) = -5

因此，C++的取余在被除数为正数时，零向和负向方向一致。而负数时则相反。
这就给循环取模类问题的运算带来的很多困难，负向取模需要写重载方法。

坑二：

避免直接用code.size()的返回值参与取余运算

    num=code.size();//数组长度=6

直接用就会这样，不明白：
-1 % code.size()=3;
-1 % 6 = -1;
-1 % num = -1;

题目描述

你有一个炸弹需要拆除，时间紧迫！你的情报员会给你一个长度为 n 的 循环 数组 code 以及一个密钥 k 。

为了获得正确的密码，你需要替换掉每一个数字。所有数字会 【同时被替换】

如果 k > 0 ，将第 i 个数字用 接下来 k 个数字之和替换。
如果 k < 0 ，将第 i 个数字用 之前 k 个数字之和替换。
如果 k == 0 ，将第 i 个数字用 0 替换。
由于 code 是【循环】的， code[n-1] 下一个元素是 code[0] ，且 code[0] 前一个元素是 code[n-1] 。

给你 循环 数组 code 和整数密钥 k ，请你返回解密后的结果来拆除炸弹！

暴力模拟+重写取模方案

思路：

  • 双层循环，每次都遍历k项并累加，添加到新数组中。
  • 负向取余部分特殊运算：负向取余结果=（零向取余的结果）+（模）
    for(int j=i-1;j>=i+k;j--){
        len=code.size();
        if(j<0)num=len+j%len;
        else num=j;

全部代码：

class Solution {
public:
    vector<int> decrypt(vector<int>& code, int k) {
        int num=0;
        int sum=0;
        int len=0;
        vector<int>res=code;
        if(k>0){
        for(int i=0;i<code.size();i++){
                sum=0;
                    for(int j=i+1;j<=i+k;j++){
                        num=(j)%code.size();
                        sum+=code[num];
                        // cout<<sum<<" ";
                    }
                res[i]=sum;
                // cout<<";"<<i<<":"<<sum<<";";
                }

        else if(k<0){
                for(int i=0;i<code.size();i++){
                sum=0;

                        len=code.size();

                        else num=j;
                        cout<<num<<" ";
                        sum+=code[num];
                    }
                res[i]=sum;
                }
            }

};

很慢，时间复杂度(n*k)

滑动窗口/前缀和方案

思路：
将原数组进行拼接：code=code+code。
保存拼接前的n=code.size()
取值：num= i+n±k;

前缀和就像移动一个[i~k]的窗口，每次减去第i项,加上k+1项

class Solution {
public:
    vector<int> decrypt(vector<int>& code, int k) {
        int n = code.size();
        vector<int> ans(n);
        if (k == 0) {
            return ans;
        }
        vector<int> s(n << 1 | 1);
        for (int i = 0; i < n << 1; ++i) {
            s[i + 1] = s[i] + code[i % n];
        }
        for (int i = 0; i < n; ++i) {
            if (k > 0) {
                ans[i] = s[i + k + 1] - s[i + 1];
            } else {
                ans[i] = s[i + n] - s[i + k + n];
            }
        }
        return ans;
    }
};

复杂度分析O（n）
