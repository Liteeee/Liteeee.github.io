操作：

两个函数的操作方法一致，区别在于内部实现上：

初始化:

unordered_map<int,string>newMap={{5,"A"},{6,"B"}};

插入.insert()/emplace():

newMap.insert(pair<int,string>(3,"C"))//pair成对插入

newMap.emplace(key,value)

insert与emplace的区别：

  • emplace不生成中间变量（在C++11 std各个结构中都是）

      例如std::vector.insert()会构造一个foo{value}中间变量

    替换方式：
    例如：

      bv.push_back(1);        // 隐式转换生成临时变量
  bv.emplace_back(1);     // 没有临时变量

  • emplace在map的使用中采用pair的形式（map特殊性）

  • pair中会构建两个tube

      pair<string, complex<double>> scp(make_tuple("hello"), make_tuple(1, 2));

    在map中,key和value会按照tuple1,tuple2进行识别。

总结：
一般只有在构建中间变量的代价较大时，才考虑使用emplace()

map:

使用红黑树进行内部实现（非严格平衡二叉搜索树）

特点：

  1. 有序，每个节点都是一个元素
  2. 操作时间与红黑树相同
  3. 通过中序遍历可实现从小到大遍历
  4. 时间复杂度低至log(n)

unordered_map：

使用哈希表进行内部实现，把关键码映射到HASH表中的位置进行访问

特点：

  1. 无序，
  2. 查找时间复杂度最低O(1)
  3. 遍历顺序与输入顺序不同
  4. 内存占用比红黑树高
