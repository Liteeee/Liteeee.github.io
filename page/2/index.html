算法 2022/9/19

按频率排序按数字降序

哈希+模拟+sort()重定

给你一个整数数组 nums ，请你将数组按照每个值的频率 升序 排序。如果有多个值的频率相同，请你按照数值本身将它们 降序 排序。

请你返回排序后的数组。
示例：

输入：[1,1,2,2,2,3,-1,99,88,88,7,12,13,12]
输出：[99,13,7,3,-1,88,88,12,12,1,1,2,2,2]

##思路：

  • map统计词频
  • 模拟（用计算机来模拟真实操作，通常需要用纸）此处用sort完成两种排序
  • sort的新用法

sort（start，end,排序方式）

其中，排序方式的参数可以是函数定义，也可以是函数调用
假设sort可以认为是排序完了之后再进行排序方式
如果a与b相同，则按a与b的降序排列（a>b）
如果a与b不相同，则按他们的值a[]升序排列(a<b)

巧妙之处在于map只用来查找，充分发挥哈希的优势,实际排序发生在nums中，也利用了vector的优势

代码:

class Solution {
public:
    vector<int> frequencySort(vector<int>& nums) {
        unoredered_map<int,int>map1;
        for(auto& i : nums){
            map1[i]++;
        }
        sort(nums.begin(), nums.end(), [&](const int a, const int b) {
            if (map1[a] != map1[b]) {
                return map1[a] < map1[b];
            }
            return a > b;
        });
        return nums;
    }
};

纯随机序列定义（白噪声序列）


白噪声序列一定是平稳序列。是最简单的平稳序列。

性质

1.纯随机性

𝜸𝒌=𝟎,∀𝒌≠𝟎

说明序列各项之间没有任何关系。

2.方差齐性

𝑫𝑿𝐭=𝜸𝟎=𝝈𝟐

序列中的每个变量的方差都相同。只有方差齐性时，最小二乘法估计的参数才是准确有效的。
异方差性质：无法满足方差齐性。
用途：检验模型的【残差】是否【方差齐性】，若不是则说明相关信息不足→非平稳随机序列分析（异方差模型）

纯随机性检验（白噪声检验）

一个纯随机序列{𝑥𝑡,𝑡=1,2,…,𝑛}的延迟K样本的ρ将服从上述分布

3.假设检验

原假設：延遲期數小於或等於m的序列值之間相互獨立

GAMES104

五层模型

功能层（Function Layer）

媒体文件居多。

资源层（Resource Layer）

作为功能层的接口。

核心层（Core Layer）

提供游戏需要的工具/方法：例如内存管理、数学模块等等，
是游戏的逻辑管理的基础。

平台层（Platform Layer）

对不同平台进行处理：不同硬件的输入方式、渲染接口、功能特性等。

建立引擎的步骤

  • 有了【平台层】，解决了不同平台特性的问题
  • 然后建立所有功能处理的内核，【核心层】
  • 再建立【资源层】对【资源进行加载和管理】
  • 接着我们建立【功能层】来进行与【游戏相关的操作】
  • 最后我们建立【工具层】将功能层进行的与游戏相关的操作【可视化为编辑器】出来供使用者们使用

中间件

1.从资源导入到游戏运行的过程

1.1 Importing

将不同的媒体数据(resource)载入为高效的数据形式(asset)
示例：

unity中：importing
创建meta文件：记录资源设置
创建引擎本地高效资源
创建关联文件：关联原始资源与引擎实际资源
GUID：资源唯一识别号

1.2 关联资产(composite asset)

将一个角色与他的资源(GUID mesh text animation等)关联起来（在内存中）

1.3 资源管理 Runtime Asset Manager

处理运行时的资源加载和卸载，把内存中的资源卸载下来，并且保持他们的关联性。
资源管理的关注面：

硬件性能：可以申请多少内存
资源回收：回收资源时的耗时
资源加载：加载耗时 - 同步异步处理

git reset --mixed : 1.移动本地版本库的指针；2.重置暂存区。（默认的参数）
git reset --hard  : 1.移动本地版本库的指针；2.重置暂存区；3.重置工作区。

配置文件

.gitignore 用于配置不需要git控制的文件 支持正则表达式

文本规范

.gitignore 的格式规范如下：
• 所有空行或者以 # 开头的行都会被 Git 忽略。
• 可以使用标准的 glob 模式匹配，它会递归地应用在整个工作区中。
• 匹配模式可以以（/）开头防止递归。
• 匹配模式可以以（/）结尾指定目录。
• 要忽略指定模式以外的文件或目录，可以在模式前加上叹号（!）取反。
# 忽略所有的 .a 文件

示例

*.a
# 但跟踪所有的 lib.a，即便你在前面忽略了 .a 文件
!lib.a
# 只忽略当前目录下的 TODO 文件，而不忽略 subdir/TODO
/TODO
# 忽略任何目录下名为 build 的文件夹
build/
# 忽略 doc/notes.txt，但不忽略 doc/server/arch.txt
doc/*.txt
# 忽略 doc/ 目录及其所有子目录下的 .pdf 文件
doc/**/*.pdf
